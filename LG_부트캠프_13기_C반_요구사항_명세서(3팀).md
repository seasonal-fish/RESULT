# LG 부트캠프 13기 C반 요구사항 명세서 (3팀)

> 프로젝트: **요즘애드 (YozumAd)**  
> 팀: LG 부트캠프 13기 C반 3팀  
> 최종 업데이트: 2026-07-01 (Confluence 동기화)

---

## 프로젝트 개요

광고 문구·포스터 이미지에 포함된 사회적·역사적 민감 표현을 AI(RAG 파이프라인)로 자동 진단하고, 실시간 신조어·밈 트렌드 데이터를 활용해 안전한 광고 카피를 생성·검토하는 서비스입니다.

---

## 프로그램 구성

| 프로그램 | 기술 스택 | 역할 |
|----------|-----------|------|
| `program_frontend` | Next.js, TypeScript, Tailwind CSS | 광고 문구 입력 UI, 위험도 리포트, 히스토리, 대시보드 |
| `program_backend`  | Go 1.26, Gin, PostgreSQL 15+pgvector, OpenAI API | RAG 파이프라인, REST API 서버 |
| `program_data`     | Python 3.12, pytest, APScheduler, Selenium | 신조어·뉴스 크롤링, DB 적재, 트렌드 동기화 |

## A. 사용자 관점 기능 요구사항 (R-01 ~ R-12)

| R-ID | 요구사항 | 시연 타임스탬프 |
|------|---------|---------------|
| R-01 | 웹에서 광고 문구를 입력하고 '검토'를 실행한다 | 01:38 |
| R-02 | 문구 내 위험 표현 구간을 심각도별(high / needs_review / low)로 하이라이트한다 | 01:45 |
| R-03 | 각 위험 판정의 근거(사유·근거 출처)를 제시한다 | 01:50 |
| R-04 | 위험 표현에 대한 대체 문구를 1개 이상 제안한다 | 02:22 |
| R-05 | 입력 문구의 위험도 점수(0~100)를 산출·표시한다 | 02:30 |
| R-06 | 포스터 이미지를 업로드하면 OCR로 이미지 속 문구를 추출한다 | 02:49 |
| R-07 | OCR로 추출한 문구를 동일한 검토 파이프라인(R-02~R-05)에 재사용한다 | 02:52 |
| R-08 | 트렌드어 기반으로 광고 문구를 생성하고 자동으로 리스크를 검토한다 | 03:53 |
| R-09 | 과거 검토 히스토리와 요약 통계를 조회한다 | 04:14 |
| R-10 | 신조어 활성도를 대시보드에서 조회한다 — 상위 활성도 신조어의 검색 트렌드 기반 활성도 추이 그래프가 표시된다 | 04:30 |
| R-11 | 민감 사건·논란 광고 사례를 대시보드에서 조회한다 | 04:45 |
| R-12 | 웹 브라우저에서 별도 설치 없이 접근·사용할 수 있다 | 00:01 |
